Worksheet 8 - Potentiometer

1. Introduction

A potentiometer is an adjustable resistor used to create a variable voltage signal, typically between 0 and 10 V. In PLC systems, it is commonly used to set speeds, adjust levels, or simulate sensor values. Understanding how potentiometers work helps engineers control analogue inputs, scale values, and test system responses accurately.

3. So What?

Why this matters

Potentiometers are used to give operators direct control over speed, pressure, temperature or position. Offsets and scaling change how inputs behave—if you don't understand them, you can misdiagnose faults. Real systems rarely use the full 0–10 V range, so correct scaling is essential.

Offset vs. scalar

Offset shifts the whole reading up or down. Scalar stretches or compresses the range. Use both to map 0–10 V to meaningful units like % or °C.

Practical testing

Set the pot to 0%, 50%, and 100% and confirm the scaled value follows. If it doesn't, adjust scalar first, then fine‑tune with a small offset.

Key Takeaways

  • Potentiometer is a 0–10 V analogue input to the PLC.
  • Offset shifts the whole range; scalar stretches or compresses it.
  • Correct scaling is needed to match real-world units (e.g., % or RPM).
  • Used to tune, simulate, and test system responses.

Questions & Answers

Question 1: What does a potentiometer do in a PLC system?

Correct Answer: A) It creates a variable voltage signal used to control things like motor speed or level settings.

Question 2: What voltage range does a potentiometer typically output?

Correct Answer: B) Between 0 and 10 volts.

Question 3: How does the motor respond as you turn the potentiometer?

Correct Answer: C) The motor speed increases as the potentiometer is turned to the right.

Question 4: What does applying an offset do to the input signal?

Correct Answer: D) It shifts the signal higher or lower to correct for drift or to fine-tune control.

Question 5: Why is scaling (scalar) important in PLC systems?

Correct Answer: B) It adjusts the signal to match real-world values, ensuring the system behaves as expected.
